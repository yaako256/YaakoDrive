# フロントエンド確認した点
## 禁止事項の確認
- 認証がなかったら中身を見れない。
- 子孫に移動できない
- 同名リネームできない。
- フォルダ作成時、同名にできない。
- 復元時に同名にできない。
- 空文字のみ、255文字を超える名前変更はできない
- ログアウト後、Cookieが残った状態でAPIをたたいて401が出るかの確認 → 正常に`401 Unauthorized`
- 別ユーザのNodeを移動/削除/ダウンロードできない。 → 正常に`404 NotFound`
- 論理削除済みNodeのリネーム/移動/ダウンロードURL発行
リネーム/移動は`409 (Conflict)`で`node is already deleted`だったが、ダウンロードURLは`404 (Not Found)`で `node not found`だったが、これは正常か
- フォルダはダウンロードURL発行できない。 → 正常に`422 (Unprocessable Entity)`で`node is not a file`

## 通常動作の確認(フロントエンド上)
- フォルダ作成
- アップロード
- ダウンロード
- フォルダ間の移動/ルートへの移動
- ネストが深いフォルダでの動作(フォルダ作成/ファイルアップロード/移動)
- ゴミ箱追加
- 空フォルダの削除/復元
- 完全削除 → 物理ファイルも消えることを確認。SQLからも消えることを確認
- 削除済みノードへのアクセス

## 異常動作
- 同一フォルダへの移動(エラーも出ず、普通に同一フォルダへ移動された。それでいいんだっけ？)
- `/ \ : * ? " < > |`などOSで問題になる文字を含む名前でフォルダ作成・アップロードができるか(`/ \`は以上文字列判定できたが、`: * ? " < > |`は貫通してその名前になれてしまった。ちなみにダウンロードすると`_`に変換された。ダウンロードはエラーにならないので、名前変更はできるが確認画面を出すとかがいいかもしれない。)
- フォルダごとゴミ箱 → 子を1つ復元　→ マイドライブのどこにも表示されないファイルが生まれる
-  子を1つゴミ箱 → 親フォルダをゴミ箱 → ゴミ箱でフォルダを開いたとき、子が別々ではなく、フォルダの中に入っている(同一時間で削除した風になってしまっている。)

## 友人へのTailscaleでの共有について
`NodeSharing`という方法がある。これは`tailnetに招待(無料枠の人数制限あり)`とは異なり、デバイス単位でtailscaleのIPアドレスを共有するものらしい。人数制限はない。
NodeShareingでデバイス単位でtailscaleのIPアドレスを共有したうえで、Access consoleで共有するポートを絞る感じだと思われる。
これをマイクラサーバなどでも使う。
多分以下のようにACLで定義する(Claudeが出しただけで、正しいかはチェックしていない)
```json
{
  "acls": [
    // 自分（オーナー）は全てアクセス可
    {
      "action": "accept",
      "src": ["autogroup:owner"],
      "dst": ["*:*"]
    },
    // 共有ユーザー（友人）はfrontendのポートのみ
    {
      "action": "accept",
      "src": ["autogroup:shared"],
      "dst": ["tag:yaakodrive-server:3000"]
    }
  ],
  "tagOwners": {
    "tag:yaakodrive-server": ["autogroup:owner"]
  }
}
```
