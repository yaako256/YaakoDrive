# YaakoDrive フロントエンド設計書

## 方針
Reactで実装する。
PC/スマホのブラウザから利用でき、スマホではPWAとしてホーム画面に追加してアプリ風に使えることを目指す。

## MVP画面
- ログイン画面
- ファイル一覧画面
- フォルダ内一覧画面
- ゴミ箱画面
- 検索結果画面
- ダッシュボード画面
- 管理者ユーザ管理画面

## MVP操作
- ログイン/ログアウト
- ファイルアップロード
- ファイルダウンロード
- フォルダ作成
- リネーム
- 移動
- ゴミ箱へ移動
- ゴミ箱から復元
- 物理削除
- 検索
- 使用容量などの確認
- 管理者によるユーザ作成

## API通信方針
Access Token / Refresh TokenはHttpOnly Cookieで管理するため、フロントエンドはトークン文字列を直接保持しない。
API通信時はCookieを送信できるようにする。

```ts
fetch(url, {
  credentials: "include",
})
```

401を受け取った場合は `/api/auth/refresh` を呼び、成功したら元リクエストを再実行する。
Refreshに失敗した場合はログイン画面へ戻す。

## 状態管理
MVPでは過度に複雑な状態管理ライブラリを前提にしない。
以下の状態を整理して扱う。

- ログイン中ユーザ
- 現在表示中フォルダ
- ファイル/フォルダ一覧
- アップロード進捗
- 選択中ノード
- 検索キーワード
- Dashboard情報

実装時に必要であれば、React Contextや軽量な状態管理ライブラリを検討する。

## アップロードUI
MVPでは複数ファイル選択と順次アップロードに対応する。
フォルダごとアップロードとチャンクアップロードはMVP後とする。

アップロード時は以下を表示する。

- ファイル名
- 進捗
- 成功/失敗
- サイズ上限エラー
- 同名ファイルがある場合の確認

## ゴミ箱UI
ゴミ箱でも通常のファイル構造に近い形で表示する。
削除済みフォルダの中身やメタデータを確認できるようにする。

復元時に同名ファイル・同名フォルダが存在する場合は、確認画面を出し、自動リネームを選択できるようにする。

## PWA方針
MVPではPWA化に必要な最小限を目指す。

- manifest
- icon
- service workerの検討
- ホーム画面追加

オフライン利用やバックグラウンド同期はMVP後とする。
